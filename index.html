The mechanism

A code is the only thing that has to travel physically

  1. 01

    Sender creates a cash-out request

    Funds go into on-chain escrow, locked behind a hashed secret code — the plaintext never touches the chain yet.

  2. 02

    Code shared with the recipient

    Sent by SMS, WhatsApp, or a phone call — however the sender already reaches their family.

  3. 03

    Recipient visits any registered agent

    A market trader, a kiosk, anyone with cash on hand and a Stellar wallet.

  4. 04

    Agent fronts cash, then claims on-chain

    The agent hands over cash, then submits the code to the contract to receive the escrowed funds themselves.

Why not just use an existing remittance app

Because "arrived" should mean
cash in hand, not a balance

Most Stellar-based payment tools solve settlement and stop there. RemitLastMile's contribution is the layer after that — turning a settled balance into physical currency through a permissionless agent network, not a licensed remittance counter.
